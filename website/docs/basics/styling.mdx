## Style via CSS

DayPicker includes a [small CSS
file](https://github.com/gpbl/react-day-picker/blob/master/packages/react-day-picker/src/style.css)
to use as template. If your build tools (like Create React App) allows to import
CSS files, import it directly in your script:

```js
import 'react-day-picker/dist/style.css';
import { DayPicker } from 'react-day-picker';
```

Each [UI element](/api/types/uielement) is described by a CSS class prefixed with `rdp-`.

```css
/* container */
.rdp {
  background: salmon:
}
/* head element */
.rdp-head {
  font-weight: bold;
  color: blue;
}
/* etc. */
```

## Custom CSS classes

Replacing the default class names is useful with [CSS
Modules](https://github.com/css-modules/css-modules) or other JS-in-CSS
bundlers. 

To use custom classes, use `classNames`. [`UIElement`](/api/types/uielement)
contains a list of the elements that is possible to change.

```jsx showOutput open=no
function App() {
  const style = `
    .salmon-head { color: salmon; }
    .purple-caption { color: purple; }
  `;
  return (
    <>
      <style>{style}</style>
      <DayPicker
        classNames={{
          head: 'salmon-head',
          caption: 'purple-caption'
        }}
      />
    </>
  );
}
```

## Inline styles

To change the appearance via inline-styles, use the `styles`
prop. [`UIElement`](/api/types/uielement) contains a list of the elements
that is possible to change.

```jsx showOutput open=no
function App() {
  return (
    <DayPicker
      styles={{
        caption: { color: 'salmon ' },
        head: { color: 'violet' }
      }}
    />
  );
}
```

## Styling modifiers

[Modifiers](../basics/modifiers) can be styled using `modifiersStyles` or `modifiersClassNames` props.
